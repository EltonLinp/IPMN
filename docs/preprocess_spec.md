# Preprocessing & Feature Extraction Specification

本阶段目标是在 Celeb-DF v2 数据集上统一生成多模态输入特征，确保后续训练阶段可以直接加载结构化结果。整体规则如下：

---

## 1. 视频管线

| 步骤                | 说明                                                                                                    |
|---------------------|---------------------------------------------------------------------------------------------------------|
| 人脸检测            | 使用轻量 RetinaFace（或同级模型）逐帧检测主脸；若同帧检测到多个人脸，以最大框 + 追踪 ID 稳定主身份。 |
| 时域跟踪            | 基于 KLT / SORT 追踪器保持 ID 一致，若跟踪失败，回退至最近一次成功检测。                               |
| 全脸裁剪            | 在检测框外额外扩展 15% 边距，裁剪后 resize → **224×224**；像素范围归一化到 `[0,1]`。                    |
| 口部 ROI            | 在检测框内提取中心下方 40% 高度、60% 宽度的区域；resize → **96×96**。                                   |
| 帧率和片段长度      | 输入视频按照 **25 fps** 解析，取连续 **2–5 秒**（50–125 帧）片段；不足时补零或镜像填充。                 |
| 数据落盘            | - `full_face/clip_id/frame_%04d.png` 保存全脸帧序列<br>- `mouth_roi/clip_id/frame_%04d.png` 保存口部。 |

> **注意**：统一以 `clip_id` 表示处理后的片段 ID，建议格式 `subject_video_clip`。

---

## 2. 音频管线

| 步骤             | 说明                                                                                                       |
|------------------|------------------------------------------------------------------------------------------------------------|
| 音轨提取         | 使用 FFmpeg（或 Librosa）从原视频中提取音频，输出单声道 PCM。                                             |
| 采样率           | 重采样至 **16 kHz**，幅度归一化到 ±1。                                                                    |
| Log-mel 特征     | 使用 25 ms 窗长、10 ms hop，生成 **80 维** log-mel 频谱；保存为 `float32` Numpy 数组 `mel.npy`。          |
| 原波形存储       | 同时保存归一化后的原始波形 `waveform.pt`（Torch tensor）或 `waveform.npy`，供自监督 / 音频增强使用。     |
| 对齐信息         | 将 mel 与视频帧时间戳对齐，记录 hop 对应的帧索引区间，便于同步模块引用。                                 |

---

## 3. 音视频同步片段

| 步骤             | 说明                                                                                          |
|------------------|-----------------------------------------------------------------------------------------------|
| 滑窗切片         | 以 **0.2–0.3 秒**（5–7 帧）为一个窗口，独立截取全脸序列、口部 ROI、对应音频帧。                |
| 索引记录         | 每个窗口生成统一的 `clip_id/window_%03d` 文件夹，包含：<br>- `frames/` 下口部帧<br>- `mel.npy` |
| 标签保存         | 记录该窗口的全局时间戳和 subject，便于后续同步增强或判别。                                     |

## 5. 样例输出

- `data/processed_samples/clip_0001/` summarises一段真实视频片段的预处理结果。  
- `data/processed_samples/clip_0002/` summarises一段伪造视频片段的结果。  

两个样例仅包含最小化的 metadata 与占位文件，用来展示最终的组织结构与字段定义，具体数值在实际批处理时生成。 这也是 downstream 组件期望读取的格式。*** End Patch
